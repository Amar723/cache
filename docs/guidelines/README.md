# Cache guidelines

`AGENTS.md` is the source of truth for agent behaviour.

Use these short guides only when the task touches that area:

- `security-guidelines.md`: auth, RLS, ownership, privacy, secrets, and logs.
- `testing-guidelines.md`: targeted checks and when to broaden coverage.
- `typescript-guidelines.md`: strict TypeScript rules for this app.
- `anti-patterns.md`: changes agents must avoid.

Do not add more guideline files unless the repo has a concrete, repeated need.
